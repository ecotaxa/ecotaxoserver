EcotaxoServer Architecture and process

Taxonomy is managed in a centralized manner on EcotaxoServer.
Below a description of the taxa lifecycle :
  • New taxa are created through local Ecotaxa instances by project managers directly on the EcotaxoServer.
  • Newly created taxa are in "Not-reviewed" status and are visible in all instances.
  • Periodically newly created taxa are reviewed then become Active.
  • The periodic review may also cause some taxon become deprecated, they will then appear with "(deprecated) " on their name.
  • When possible deprecated taxa will be tagged to be renamed by a new taxon automatically.
  • Once a deprecated is not used anymore in any instance, it will be destroyed.
Ecotaxa instances will synchronize periodically.
UniEuk update are done manualy by EcoTaxoServer administrators
